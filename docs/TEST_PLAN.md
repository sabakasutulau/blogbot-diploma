# План тестирования

## Цель тестирования

Проверить, что система корректно создает, хранит, публикует и отображает записи блога, а административные действия защищены токеном.

## Проверяемые компоненты

1. REST API.
2. Публичная веб-страница.
3. Административная панель.
4. Telegram-бот.
5. Работа с базой данных.

## API-тесты

| № | Проверка | Ожидаемый результат |
|---|---|---|
| 1 | `GET /api/health` | Ответ `status: ok` |
| 2 | `POST /api/posts` без токена | Ошибка 401 |
| 3 | `POST /api/posts` с токеном | Создан пост со статусом draft или published |
| 4 | `GET /api/posts` | Возвращается список публикаций |
| 5 | `PATCH /api/posts/{id}/publish` | Статус меняется на published |
| 6 | `DELETE /api/posts/{id}` | Пост удаляется |

## Ручные тесты интерфейса

| № | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Открыть `/` | Отображается публичная лента |
| 2 | Открыть `/admin` | Отображается форма управления публикациями |
| 3 | Создать пост через админ-панель | Пост появляется в списке |
| 4 | Опубликовать пост | Пост становится доступен на главной странице |
| 5 | Открыть ссылку поста | Загружается страница публикации |

## Тесты Telegram-бота

| № | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Отправить `/start` | Бот выводит справку |
| 2 | Отправить `/newpost` | Бот запрашивает заголовок |
| 3 | Ввести слишком короткий заголовок | Бот просит исправить данные |
| 4 | Ввести корректный заголовок и текст | Создается черновик |
| 5 | Нажать «Опубликовать» | Запись получает публичную ссылку |
| 6 | Отправить `/drafts` | Бот показывает черновики пользователя |

## Автоматизация

В проекте подготовлены тесты `pytest`:

```bash
pytest
```

Также подготовлена коллекция `postman_collection.json`, которую можно импортировать в Postman и использовать для демонстрации API-запросов.
