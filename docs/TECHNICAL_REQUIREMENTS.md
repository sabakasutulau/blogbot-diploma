# Технические требования к программному продукту

## 1. Назначение продукта

Программный продукт предназначен для создания, хранения, публикации и просмотра записей блога с использованием Telegram-бота, REST API и веб-интерфейса.

## 2. Требования к функциональным характеристикам

Система должна обеспечивать:

1. Создание публикации через Telegram-бота.
2. Создание публикации через веб-панель администратора.
3. Сохранение публикаций в базе данных.
4. Разделение публикаций по статусам: черновик, опубликовано, архив.
5. Публикацию черновиков.
6. Отображение опубликованных материалов в публичной ленте.
7. Просмотр отдельной публикации по ссылке.
8. Удаление публикаций администратором.
9. Предоставление REST API для работы с публикациями.
10. Проверку состояния сервиса через endpoint `/api/health`.

Требования к временным характеристикам специально не предъявляются. Для локальной демонстрации основные страницы и API-запросы должны открываться без заметной задержки для пользователя.

## 3. Требования к надежности

1. Данные публикаций должны сохраняться в базе данных.
2. Административные операции должны быть защищены токеном.
3. Вводимые данные должны проходить валидацию.
4. При ошибочных запросах API должен возвращать корректные HTTP-статусы.
5. При отсутствии Telegram-токена бот не должен запускаться с неясной ошибкой, а должен сообщать о необходимости настройки `BOT_TOKEN`.

## 4. Условия эксплуатации

Продукт рассчитан на эксплуатацию на персональном компьютере или сервере с установленным Python 3.11+ и доступом к сети интернет. Для полноценной эксплуатации рекомендуется использовать PostgreSQL.

## 5. Требования к составу технических средств

Минимальная конфигурация:

- процессор: 2 ядра;
- оперативная память: 2 ГБ;
- свободное место на диске: 500 МБ;
- подключение к интернету;
- современный браузер.

## 6. Требования к информационной и программной совместимости

Используемые технологии:

- Python 3.11+;
- FastAPI;
- SQLAlchemy;
- Aiogram 3;
- PostgreSQL или SQLite;
- HTML, CSS, JavaScript;
- Uvicorn;
- Pytest;
- Postman.

## 7. Требования к программной документации

В состав документации должны входить:

1. README с инструкцией запуска.
2. Технические требования.
3. Пользовательские сценарии.
4. Описание архитектуры.
5. План тестирования.
6. Инструкция установки и внедрения.
7. Перечень исходных файлов.

## 8. Порядок контроля и приемки

Приемка выполняется по контрольным сценариям:

1. Запуск веб-приложения.
2. Открытие публичной ленты.
3. Создание записи через админ-панель.
4. Публикация записи.
5. Просмотр опубликованной записи.
6. Создание черновика через Telegram-бота.
7. Проверка API через `/docs` или Postman.
8. Запуск автоматических тестов.
