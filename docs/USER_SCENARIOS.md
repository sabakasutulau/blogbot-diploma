# Пользовательские сценарии

## Участники системы

1. **Автор блога** - создает записи через Telegram-бота.
2. **Администратор** - управляет публикациями через веб-панель.
3. **Читатель** - просматривает опубликованные материалы на сайте.
4. **Система** - сохраняет данные, проверяет права доступа и отображает публикации.

## Сценарий 1. Создание черновика через Telegram-бота

1. Автор открывает Telegram-бота.
2. Автор отправляет команду `/newpost`.
3. Бот запрашивает заголовок.
4. Автор вводит заголовок.
5. Бот запрашивает текст публикации.
6. Автор вводит текст.
7. Система сохраняет запись со статусом `draft`.
8. Бот предлагает опубликовать запись или оставить ее черновиком.

## Сценарий 2. Публикация записи через Telegram-бота

1. Автор создает черновик.
2. Бот показывает кнопку «Опубликовать».
3. Автор нажимает кнопку.
4. Система меняет статус записи на `published`.
5. Бот отправляет автору ссылку на опубликованную запись.

## Сценарий 3. Управление публикациями через админ-панель

1. Администратор открывает `/admin`.
2. Администратор вводит токен.
3. Администратор заполняет заголовок и текст.
4. Администратор выбирает, публиковать запись сразу или сохранить как черновик.
5. Система создает запись.
6. Администратор может опубликовать или удалить запись.

## Сценарий 4. Просмотр блога читателем

1. Читатель открывает главную страницу.
2. Система показывает список опубликованных записей.
3. Читатель открывает отдельную публикацию.
4. Система отображает заголовок, дату публикации и текст.

## Сценарий 5. Проверка API разработчиком

1. Разработчик открывает `/docs`.
2. Разработчик изучает список API-методов.
3. Разработчик отправляет тестовые запросы.
4. Система возвращает ответы в формате JSON.
