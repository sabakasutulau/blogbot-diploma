# Чек-лист соответствия проекта требованиям диплома

Документ фиксирует, какие элементы практической части уже закрыты проектом.

## Функциональность

| Требование | Реализация в проекте |
|---|---|
| Создание публикаций | Telegram-бот `/newpost`, веб-панель `/admin`, API `POST /api/posts` |
| Хранение данных | Таблица `posts` в базе данных через SQLAlchemy |
| Публикация материалов | Кнопка публикации в Telegram-боте, API `PATCH /api/posts/{id}/publish`, кнопка в админ-панели |
| Публичный просмотр | Главная страница `/`, страница поста `/post/{slug}` |
| Администрирование | Панель `/admin`, защищенные API-запросы через `X-Admin-Token` |
| REST API | OpenAPI-документация FastAPI на `/docs` |
| Тестирование API | `tests/test_api.py`, Postman-коллекция |
| Проверка работоспособности | `GET /api/health` |

## Технологические требования

| Область | Реализация |
|---|---|
| Верстка | HTML-шаблоны Jinja2, CSS, адаптивная сетка |
| Логика приложения | Python, FastAPI, Aiogram, JavaScript в админ-панели |
| База данных | SQLite для быстрой демонстрации, PostgreSQL через Docker Compose |
| API | REST endpoints в `app/main.py` |
| Тестирование | Pytest, Postman |
| Документация | README и материалы в папке `docs/` |

## Что еще нужно сделать перед сдачей

1. Запустить проект локально и сделать скриншоты: главная страница, страница поста, админ-панель, `/docs`, Telegram-бот.
2. Подключить реальный токен Telegram-бота.
3. Заполнить базу несколькими демонстрационными публикациями.
4. Снять короткое видео-демо на 1-2 минуты.
5. Перенести нужные материалы в шаблоны ПЗ и приложений.
