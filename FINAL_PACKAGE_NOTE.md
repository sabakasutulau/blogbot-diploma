# FINAL PACKAGE NOTE — BlogBot Diploma

Это чистая упаковка проекта для передачи, загрузки в репозиторий и подготовки к защите.

## Что исключено из архива

- `.env` — локальные секреты и токены;
- `.venv/` — локальное виртуальное окружение;
- `.pytest_cache/` — кэш тестов;
- `__pycache__/` и `*.pyc` — скомпилированные Python-файлы;
- `*.db`, `*.sqlite`, `*.sqlite3` — локальные базы данных.

## Что нужно создать локально после распаковки

```powershell
Copy-Item .env.example .env
```

В `.env` вручную заполнить:

```env
BOT_TOKEN=токен_из_BotFather
TELEGRAM_CHANNEL_ID=@username_канала
BOT_OWNER_IDS=ваш_telegram_user_id
```

`BOT_OWNER_IDS` ограничивает управление ботом владельцем. Если оставить пустым, бот работает в режиме разработки и доступен всем пользователям.

## Основной режим работы

Проект теперь поддерживает Telegram-first сценарий:

1. Пользователь создаёт публикацию через Telegram-бота.
2. Бот сохраняет публикацию в базу данных.
3. Бот показывает предпросмотр и кнопки управления.
4. Основной вариант публикации — отправка поста в Telegram-канал.
5. Сайт остаётся рабочей дополнительной частью проекта и может использоваться для демонстрации.
